# Brag Document Skill

A Claude Code skill that automatically collects your contributions from GitHub, Jira, and Linear, generates impact-focused descriptions, and saves them as a brag document you can use for performance reviews.

## What it does

- Pulls data from multiple sources (PRs merged, reviews done, tickets resolved, issues closed)
- Generates impact-focused narratives using Claude (not just "what" you did, but "why" it mattered)
- Classifies entries into categories (Features, Bug fixes, Code reviews, Leadership, Process improvements)
- Saves entries in Markdown files organized by configurable time periods (daily, weekly, monthly, quarterly)
- Generates consolidated summaries for performance review prep

## Installation

Copy the `.claude/skills/brag/` directory into your project's `.claude/skills/` folder.

## Usage

### Initial setup

```
/brag init
```

Walks you through configuring:
- Time period granularity (daily/weekly/monthly/quarterly)
- Data sources (GitHub, Jira, Linear) and connection type (MCP or API)
- Impact categories (defaults provided, fully customizable)
- Optional automated schedule

### Collect entries

```
/brag collect --period last-week
/brag collect --period yesterday
/brag collect --period last-month
/brag collect --period 2026-03-01:2026-03-15
```

Pulls data from your configured sources for the given period, generates impact descriptions, classifies them, and appends to the correct period file.

### Generate summary

```
/brag summary --range 2026-Q1
/brag summary --range 2026-01:2026-03
```

Reads all period files in the range and generates an executive summary — ready for performance reviews.

## Data sources

| Source | What it collects | Connection |
|--------|-----------------|------------|
| GitHub | PRs merged, PR reviews, issues closed | MCP or API (`GITHUB_TOKEN`) |
| Jira | Tickets resolved/closed | MCP or API (`JIRA_API_TOKEN`) |
| Linear | Issues completed | MCP or API (`LINEAR_API_KEY`) |

## Output example

```markdown
# Brag Document - 2026-W14

## Features entregues
- **Implemented distributed cache system** (PROJ-123) - Reduced API latency by 40% by introducing Redis as a caching layer between the service and the database. [PR #45](link)

## Bug fixes
- **Fixed critical crash in payment flow** (PROJ-456) - Resolved null pointer exception affecting checkout in production. [PROJ-456](link)

## Code reviews
- **Reviewed OAuth2 auth migration** (PR #78) - Identified security vulnerability in refresh token flow before merge. [PR #78](link)
```

## Configuration

Generated by `/brag init` at `brag.config.yaml`:

```yaml
period: weekly
output_dir: brag
categories:
  - Features entregues
  - Bug fixes
  - Code reviews
  - Lideranca tecnica
  - Melhorias de processo
  - Outros
sources:
  github:
    type: mcp
    username: your-username
    repos: all
  jira:
    type: mcp
    project_keys: all
```
